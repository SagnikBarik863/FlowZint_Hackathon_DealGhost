How should the demo/admin split view be accessed?

Customer always sees clean chat only — this controls how judges & FlowZint staff reach the split panel during the demo

A

Separate /demo route

Judges go to /demo — shows full split panel (chat left, intelligence right). Customer goes to /chat — sees only the conversation. Clean separation, zero auth needed.

✓ Simplest to build
✓ Best for live demo
→ Recommended
B

Query param toggle

/chat?view=admin expands the split panel inline. Same page, URL-controlled. Customer at /chat sees only chat.

✓ One page to maintain
✗ More complex logic
C

Hidden keyboard shortcut

Customer sees clean chat. Presenter presses a key combo (e.g. Ctrl+Shift+D) on demo day to reveal the intelligence panel sliding in from the right.

✓ Most impressive live demo effect
✗ Risky if shortcut misfires
D

Separate /admin dashboard

Full admin dashboard at /admin with multiple leads, analytics, and the split panel as one feature. More like a real product's backend.

✓ Most "real product" feel
✗ More scope, 4-week risk
