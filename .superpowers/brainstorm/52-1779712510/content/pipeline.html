Section 2 — The 6-Layer Intelligence Pipeline

Every user message flows through all 6 layers. This is the brain of DEALGHOST.

💬 User Message + Conversation History + Current State
▼
L1 · Conversational Understanding
CLAUDE SONNET
→ What is the client actually saying beyond surface words?
→ Detect: corrections, contradictions, removals, elaborations
→ Identify: business domain, urgency signals, workflows hinted at
→ Extract: key entities (systems, people, processes)
OUT: SemanticUnderstanding { intent, domain, corrections[], contradictions[], entities[] }
L2 · Canonical Requirement Extraction
CLAUDE SONNET
→ Extract features, integrations, platforms, constraints
→ Each feature gets a confidence score (0–1)
→ Map raw text → canonical IDs via pgvector similarity
   "live tracking" → realtime_delivery_tracking
   "GPS tracking" → realtime_delivery_tracking
→ Distinguish: confirmed fact vs. inferred assumption
OUT: ExtractionResult { features[], integrations[], confidence{}, assumptions[] }
L1 + L2 run in parallel — both fire simultaneously to save latency
▼
L3 · State Intelligence Engine
TYPESCRIPT (deterministic) + CLAUDE (for contradiction resolution only)
→ Merge L1 + L2 outputs into ProjectRequirementState
→ Resolve contradictions (call Claude if needed)
→ Track confidence score per field
→ Separate confirmed facts from assumptions
→ Flag ambiguities for follow-up targeting
→ Compress old conversation into rolling summary
→ Recalculate completeness score (weighted)
→ Rank discovery targets by proposal-blocking impact
OUT: Updated ProjectRequirementState + DiscoveryTargets[] + Contradictions[] + Ambiguities[]
▼
L4 · Discovery Strategy Engine
CLAUDE SONNET
→ Reads updated state + discovery targets + contradiction flags
→ Asks: "What is most valuable to learn RIGHT NOW?"
→ Strategies: clarify_scope · probe_complexity · resolve_contradiction
              confirm_assumption · discover_workflow · offer_summary
→ Generates architect-quality question — not chatbot filler
→ Decides: should we offer a summary/proposal instead of asking?
OUT: { strategy, targetField, reasoning, question, readyForSummary: bool }
L5 · Lead Intelligence
CLAUDE SONNET
→ Business maturity signals
→ Budget realism check
→ Project clarity score
→ Urgency & intent signals
→ Technical feasibility
→ Engagement depth
→ Commercial fit for FlowZint
OUT: LeadScore { score, label, breakdown{}, narrative }
L4 + L5 run in parallel — both read from L3 output simultaneously
▼
L6 · Proposal Intelligence  (on-demand only — not every turn)
CLAUDE SONNET · Multi-step reasoning chain
Step 1 — Architecture Selection
  What tech stack fits this project?
  Why? What are the trade-offs?

Step 2 — Complexity Calibration
  Realistic hours per feature
  Phasing strategy
Step 3 — Risk Identification
  Technical risks, scope risks
  Budget risks, timeline risks

Step 4 — Proposal Assembly
  Full structured proposal
  Executive summary → pricing
OUT: ProposalContent { executiveSummary, scope, phases, pricing, techStack, risks, team, assumptions }
▼
🌊 SSE Stream → Next.js UI (layer-by-layer as pipeline runs)
⚡ PRE-FLIGHT · Intent Classification  —  fires BEFORE L1, costs almost nothing
GROQ Llama-3.1-8b  ·  Input: ~200 tokens  ·  Output: 1 label  ·  Latency: ~300ms
Routes: COLLECTING_INFO → full pipeline  |  REQUESTING_DONE → skip to summary  |  READY_FOR_PROPOSAL → trigger L6

Execution timing (typical turn)

0ms     → Intent classification fires (Groq)
300ms   → L1 + L2 fire in parallel (Claude)
4–8s    → L1 + L2 complete, L3 runs instantly (TypeScript)
8s      → L4 + L5 fire in parallel (Claude)
10–14s  → L4 question starts streaming to UI ← user sees response
14s     → L5 score saved, state persisted to Redis + Supabase
