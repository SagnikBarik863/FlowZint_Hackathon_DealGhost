Biggest Architectural Risks
⚠ Anthropic rate limits (Tier 1)
New API keys are rate-limited. Mitigated by prompt caching and keeping tokens lean. Upgrade tier if needed.
⚠ pgvector raw SQL migration
Prisma doesn't support VECTOR type natively. The migration file needs a raw SQL block. I'll generate this exactly — just needs one careful run.
⚠ SSE through Vercel proxy
Vercel can buffer SSE responses. Needs explicit headers: Cache-Control: no-cache, X-Accel-Buffering: no. One config line — easy to miss.
⚠ Rolling summary losing context
If the summarisation prompt is weak, important facts get lost after turn 8. The summary prompt is critical — needs explicit instruction to preserve all extracted facts.

What this redesign fixes vs. current implementation

NOW: Single-shot extraction, misses fields, no confidence  →  L1+L2 parallel reasoning chain, confidence per field
NOW: "live tracking" ≠ "GPS tracking" → two features  →  Both → realtime_delivery_tracking via canonical mapping
NOW: No contradiction detection → corrupted state  →  L1 detects, L3 flags, L4 resolves

NOW: Message count gates for scoring → weak  →  Claude business reasoning → budget realism, feasibility
NOW: Single-shot proposal → shallow output  →  4-step reasoning chain → architecture + risks + pricing
NOW: Full history sent every turn → unbounded tokens  →  Rolling summary after turn 8 → stable token budget
NOW: 30s Vercel timeout risk on deep chains  →  Hono on Railway → no timeout, proper SSE streaming
