Feature Ontology  — pgvector table in Supabase
Solves the core deduplication problem: "live tracking", "GPS tracking", "track orders" all resolve to the same canonical feature. No more duplicate features in the extracted state.
feature_ontology table
id              TEXT PRIMARY KEY     -- 'realtime_delivery_tracking'
canonical_name   TEXT                -- 'Real-time Delivery Tracking'
category         TEXT                -- 'logistics', 'auth', 'payments'
aliases          TEXT[]               -- ['live tracking', 'GPS tracking', ...]
embedding        VECTOR(1536)         -- semantic similarity search
typical_complexity TEXT                -- 'LOW', 'MEDIUM', 'HIGH'
typical_hours     INT4RANGE            -- [40, 80]
dependencies     TEXT[]               -- ['user_auth', 'maps_integration']
incompatible_with TEXT[]               -- features that conflict
How canonical mapping works at extraction time:
1. L2 extracts raw feature text: "I need live GPS tracking for drivers"
2. Embed the text → 1536-dimension vector
3. pgvector cosine similarity search against feature_ontology
4. Score > 0.85 → map to realtime_delivery_tracking (confident match)
5. Score 0.60–0.85 → Claude confirms: "Is this the same as real-time delivery tracking?"
6. Score < 0.60 → new canonical entry created, added to ontology
7. Feature stored with canonicalId + confidence + rawText preserved

What this adds vs. the current implementation

Current: Features stored as raw strings — "live tracking" and "GPS tracking" become two separate features

New: Both resolve to realtime_delivery_tracking via pgvector. One feature, no duplicates.

Current: No confidence tracking — every extracted field treated as equally certain

New: Every field has a confidence score. Low-confidence fields get re-targeted by the discovery engine.

Current: No contradiction detection — if client changes their mind, old data persists silently

New: L1 detects contradictions, L3 flags them, L4 resolves them before they corrupt the proposal.

Current: Full conversation history sent every turn — grows unbounded

New: Rolling summary compresses turns older than 8. Claude always has context without burning tokens.
