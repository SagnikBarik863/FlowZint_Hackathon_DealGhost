Migration steps you run (I'll generate the exact SQL):
1. Enable pgvector extension in Supabase dashboard → Extensions → enable "vector"
2. prisma migrate dev → adds new columns to ProjectAnalysis + creates FeatureOntology table
3. Run raw SQL to add embedding VECTOR(1536) column + index to feature_ontology
4. Run ontology seed script → embeds 150 pre-built canonical features, inserts into DB
5. Done — zero manual DB work after this
🔢 Embedding model for feature ontology
OpenAI text-embedding-3-small  ·  1536 dimensions  ·  $0.02 per 1M tokens
Seeding 150 features ≈ 7,500 tokens ≈ $0.00015 total (one-time, basically free)
Per new feature at runtime ≈ ~50 tokens ≈ $0.000001 per embedding call
Add OPENAI_API_KEY to your .env — only used for embeddings, nothing else

What you handle vs. what gets built for you

You handle: Enable pgvector in Supabase dashboard · Set env vars (ANTHROPIC_API_KEY, OPENAI_API_KEY, GROQ_API_KEY, DATABASE_URL, REDIS_URL) · Railway deploy · Git pushes

Built for you: All migration files · Seed script · Every pipeline layer · All prompts · State management · Redis cache layer · SSE streaming · Frontend proxy · Proposal generation
