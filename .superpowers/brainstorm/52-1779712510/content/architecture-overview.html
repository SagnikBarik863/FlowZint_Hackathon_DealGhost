Key architectural decisions

  • No serverless for AI: The Hono service runs on a persistent server (Railway). Deep multi-step Claude chains need 15–30 seconds — that kills Vercel functions.
  • Redis hot cache: Between turns, state is loaded from Redis (not Postgres) — sub-10ms vs 50–200ms. Postgres is the source of truth; Redis is the speed layer.
  • SSE streaming: The pipeline streams layer-by-layer results back to the UI. The client sees "Extracting requirements..." → "Detecting gaps..." → "Generating question..." in real time.
  • pgvector feature ontology: A pre-built table of canonical features with vector embeddings. "live tracking" and "GPS tracking" map to the same canonical ID via cosine similarity search — no more duplicate features.
  • Prompt caching on Claude: System prompts are marked as cacheable. 75% discount on repeated tokens. The extraction system prompt (~800 tokens) is reused every turn.
