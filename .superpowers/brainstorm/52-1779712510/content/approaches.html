Three Architecture Approaches for DEALGHOST

Choose the direction before we design the full system

A

Multi-Step Pipeline in Next.js

Upgrade in place. Keep Next.js as the only backend. Redesign each pipeline stage as multi-step Claude reasoning chains. Replace single-shot prompts with structured reasoning. Add proper state machine, confidence scoring, and canonical feature mapping — all inside the existing app.


Stack: Next.js 16 · Claude Sonnet · Groq · Supabase · Prisma · Vercel


Pros

  • Fastest to implement (~1–2 weeks)
  • No new infrastructure
  • Single deployment on Vercel
  • Familiar codebase

Cons

  • Vercel 30s timeout limits pipeline depth
  • Multi-step Claude chains risky in serverless
  • No streaming-first architecture
  • All pipeline stages compete for one request window
B

Dedicated AI Service + Next.js Frontend ⭐ Recommended

Proper separation. Next.js becomes the UI layer only. A dedicated Node.js/Hono AI service owns the entire intelligence pipeline — no timeout constraints, proper streaming, complex multi-step reasoning chains, Redis hot state cache, and an extensible orchestration engine. Deployed on Railway or Render.


Stack: Next.js (UI) · Hono (AI service) · Claude Sonnet · Groq · Supabase + pgvector · Redis · Railway/Render


Pros

  • No timeout constraints on deep reasoning chains
  • Proper SSE streaming to the UI
  • Redis for hot state between turns (sub-10ms reads)
  • pgvector for semantic feature canonicalization
  • Clean separation: UI vs. intelligence
  • Independently scalable AI pipeline

Cons

  • Two services to deploy and manage
  • Slightly more infrastructure (Redis, Railway)
  • ~2–3 weeks to implement correctly
C

Mastra Agent Architecture

Maximum power. Use Mastra as a full agent orchestration framework. Claude drives the entire conversation as a proper agent with tool calls — each pipeline stage becomes a tool (extract requirements, score lead, generate proposal). Claude decides what to invoke and when. Full agent memory, step tracing, and workflow graphs.


Stack: Mastra · Claude Sonnet (tool-calling agent) · Groq · Supabase + pgvector · Redis · Railway


Pros

  • Claude autonomously drives discovery — deeply adaptive
  • Tool-calling is the cleanest pattern for complex pipelines
  • Built-in step tracing and observability (Mastra)
  • Most sophisticated conversational intelligence possible

Cons

  • Mastra is production-new — less stable
  • Hardest to debug when Claude makes wrong tool decisions
  • Most expensive (more Claude tokens per turn)
  • 3–4+ weeks to implement properly
  • Overkill for single-agency internal use
